## 迎接挑戰

現在，我想交給你一個挑戰，這個挑戰是幫助我架起一個 LAMP (Linux Apache MySQL PHP) 架構的開發環境，想一下，你會怎麼做？

想好了嗎？讓我猜一下

你可能會先確定這個 Linux 平臺是哪個 distribution，ubuntu、CentOS 或 alpine 等等，之後你可能會開始下關鍵字，例如「ubuntu install MySQL」，並根據教學裝好這些東西，當中你可能還會遇到某些環境上的 bug 讓你無法以網路上寫的安裝步驟正常的進行安裝

通常這類環境佈建的過程會耗得很久，很多人原本寫程式的興致都被這準備工作所破壞

除此之外，如果你在你的 local 主機上完成了你的程式開發，現在你想部署到雲端上，這時的你又得再頭痛一次，因為

- 雲端主機可能和你本地主機作業系統不同
- 所有環境你必須重新佈建一次（重新下關鍵字，看看怎麼裝 mysql…😔）

這些痛，docker 都聽到了，看看他如何解決這些問題。

## Docker 救世主

為了方便理解，你現在可以先將 Docker 理解成一個 Virtual Machine（以下簡稱 VM）。

所謂的 VM 就是在你的電腦中開啟另一個虛擬環境，這個環境可以是不同的作業系統，而你可以到這個環境去進行任何操作，甚至你可以完全在 VM 上進行你的開發工作，如同操作你原本的作業系統一樣。

想像一下，如果你的 VM 是可以「拆裝的」，你可以將那個上面有作業系統、環境設定和程式碼的 VM 整個拆下來，然後裝到雲端主機去，那麼前述提到的搬遷問題不就都解決了嗎？此時的你不用再裝環境，不用怕不相容，因為你是把整個 VM 包括作業系統一起搬進去。

現在搬遷問題解決了，我們再看看最初提到的環境佈建問題（架設 LAMP）

你可以想像有人幫你把這些「可拆裝的」VM 通通丟到一個網路上的大倉庫裡，這個倉庫裡頭包括「作業系統是 ubuntu，裡頭只有裝 MySQL」的 VM，「作業系統是 alpine，裡頭只有裝 PHP」的 VM 等等形形色色的 VM 們，此時如果你想要佈建一個新環境，你要做的就是把別人包好的這些 VM 通通抓下來跑在自己的電腦裡，因為不管是 MySQL 或是 PHP 都跑在自己的 VM 中，所以根本不會有和你現有環境不相容的問題

你可能會問，他們都在各別的 VM，那他們之間該如何溝通？這你不用擔心，因為 docker 都已經為你做好了

在這裡必須要趕快澄清「Docker 並不是 VM」，他只是一個 process，正因為如此，他比真的 VM 更具有優勢，他體積更小、運行速度也更快。

## Docker 是什麼

你現在對 Docker 已經有初步的理解，接下來我們要來正式看看 Docker 官網是怎麼介紹 Docker 的

> Docker 是一門技術，幫助開發者或系統管理者利用「容器」的方式 build, share 和 run 你的應用程式

「容器」在某個概念上就是把應用程式「包起來」的感覺，你會把作業系統、環境變數、程式碼、以及要執行你的程式所必備的一切元素通通包起來，類似於前面所講的 VM 的例子，而把應用程式「包起來」的過程就是所謂的「容器化」
